Simply Static Pro

27/11/2022 02/12/2022

Simply Static là 1 plugin để chuyển WordPress về dạng tĩnh (html), ở bài này mình sẽ thử trên bản Simply Static Pro, với rất nhiều tính năng tự động, giúp việc cài đặt đơn giản hơn

Cài đặt

  1. Download bản FREE Simply Static v2.1.9
  2. Cập nhập bản Simply Static Pro (có thể google bản FREE nếu bạn không muốn trả tiền)

Phiên bản Pro mới nhất ở thời điểm mình viết bài này là v1.2.3, nó có 1 tính năng mới là

  • trigger repository_dispatch GitHub actions by default after all files committed

Nôm na từ v1.2.2.1 về sau, tác giả đã tích hợp bản Pro với Github hoàn toàn tự động, bạn không cần phải thêm hay sửa bất cứ dòng code nào

Mỗi cái không rõ có phải do tác giả “đang” nâng cấp thêm nhiều tính năng quá không, mà mình thử bản mới nhất v1.2.3, cho xuống bản 1.2.2.4 hay 1.2.2 đều thấy lỗi đủ các thứ, kiểu khi đồng bộ lên Github ở bản v1.2.3 không rõ vì lý do gì tổng file tĩnh khi tạo ra chỉ khoảng 900, mà nó upload hơn 7000 file lên Github luôn, chưa kể nó cực kì chậm, chạy cả ngày vẫn chưa đồng bộ xong

Có thể 1 thời gian sau, khi tác giả cập nhập lên 1.24 hay 1.2.5 gì đó có thể nó sẽ chạy hoàn toàn tự động và mượt mà hơn, còn giờ thì tạm ở bài này, mình sẽ viết theo Simply Static Pro v1.2.1, bản dùng mình thấy ít lỗi nhất 

Cấu hình General

Destination URLs -> Use relative URLs (Sử dụng URL tương đối để có thể phù hợp với mọi domain, subdomain khác nhau)

Delivery Method: chọn Github

Ấn SAVE lại

Cấu hình Deployment

Kéo xuống phần GitHub, điền các thông tin tài khoản GitHub của bạn vào

GitHub User / Organization: điền user name 

GitHub E-Mail: điền email lúc đăng kí

Personal Access Token: tạo 1 token tại đây

Name your repository: điền chính xác GitHub User.github.io (cụ thể nick mình đặt là bibica-net thì repository điền  bibica-net.github.io)

Visiblity of your repository: để public mặc định

Name your branch: điền gh-pages

Webhook URL: nếu bạn mặc định chỉ chứa file tĩnh ở Github thì không cần điền dòng này, còn chứa file ở các dịch vụ khác thì điền vào, cụ thể ở đây mình thử dùng thêm ở Cloudflare Pages nên điền Webhook từ Cloudflare 

Điền xong các thông tin thì ấn vào SAVE CHANGES, sau đó ấn ADD REPOSITORY

Link to your repository sẽ được tạo ra là: github.com/bibica-net/bibica-net.github.io

Kích hoạt GitHub Actions với Webhook

Sử dụng Code Snippets tạo 1 file php với nội dung:

add_filter( 'ssp_webhook_args', function( $args ) {
  // Get access token.
  $options       = get_option( 'simply-static' );
  $access_token = $options['github-personal-access-token'];

  // Setup body.
  $body = array(
    'event_type' => 'repository_dispatch',
  );

  // Modify arguments for POST request.
  $args = array(
    'headers' => array(
      'Authorization' => 'token ' . $access_token,
      'Accept'        => 'application/json',
    ),
    'body' => wp_json_encode( $body )
  );
  return $args;
});

Chọn Run snippet everywhere như mặc định

Các setting và tab khác để mặc định, tạm mình cũng chưa tìm hiểu hết các tab liên quan tới search, contact, comment …. sẽ bổ xung sau

Generate

Ấn vào Generate Static Files để tạo các file tĩnh (html), sau khi tạo xong nó sẽ được tự động chuyển về repository GitHub bạn set ở trên

Ở phiên bản cũ v1.2.1 thao tác tạo file tĩnh, upload lên GitHub rất nhanh, cụ thể như thèng bibica.net khoảng 900 file tĩnh, đồng bộ mất 20-30 phút, mà ở bản mới nhất v1.2.3, không rõ vì sao lại siêu chậm, chạy cả ngày chưa xong

Cấu hình domain riêng trên GitHub

Đi tới repositories: github.com/bibica-net/bibica-net.github.io

Ấn tiếp vào Settings -> Pages -> Custom domain, điền sub domain của bạn vào, cụ thể mình dùng static.bibica.net trên GitHub

Tạo CNAME static trỏ về bibica-net.github.io (thay bằng tên tài khoản của bạn vào)

Làm đúng trình tự như thế thì gần như nó check DNS mất tầm 3-5s là domain đã có thể truy cập được

Về SSL thì nó khá dễ gây lỗi, đa phần nếu bạn quản lý qua Cloudflare mà bật đám mây vàng là Let's Encrypt không check được IP thực -> gây lỗi 😀

Tắt proxy trên Cloudflare đi thì tầm 2-3 phút là cài xong ssl thôi

Cấu hình Cloudflare Pages

Vào Cloudflare Pages, tạo project mới từ Git, nó sẽ tự kết nối sang GitHub, chọn repositories bibica-net.github.io tạo ra ban nãy, ngồi chờ ít phút để nó đồng bộ lần đầu

Tắt Automatic deployments

Settings -> Builds & deployments

Configure Production deployments: bỏ tùy chọn Enable automatic production branch deployments

Configure Preview deployments: chọn None (Disable automatic branch deployments)

Tạo Deploy hooks

Settings -> Builds & deployments -> Deploy hooks -> Add deploy hook

Đặt tên là Deploy hoặc 1 tên tùy thích, làm xong sẽ cho ra cái URL để điền vào Deployment Webhook URL phần cấu hình của Simply Static Pro

Phần Tắt Automatic deployments và tạo Webhook trên Cloudflare Pages cực kì quan trọng, không làm thì mỗi khi bên GitHub có cập nhập, nó tự kéo sang Cloudflare Pages tới mức Cloudflare khóa tài khoản luôn, phần này mình cũng thấy tác giả ghi thế thì chép lại thôi, chứ cũng không rõ có hiệu quả không nữa

Cấu hình domain riêng trên Cloudflare Pages

Custom domain trên Cloudflare thì chỉ điền subdomain sau đó cập nhập CNAME lại, Cloudflare ghi cụ thể và cập nhập nhanh hơn GitHub nhiều lắm, nếu domain quản trị trên Cloudflare thì họ tự đổi CNAME hộ bạn luôn, tầm 2-3s là đã Active 😀

Bên Cloudflare thì mình dùng subdomain là cloudflare.bibica.net

Tổng thể về cơ bản set ban đầu theo hướng dẫn của tác giả là thế 😀 khá mất thời gian nếu bạn là người lần đầu tiên làm chuyện này :((

Cấu hình Simply Static tự chạy hàng ngày 

Sử dụng Code Snippets tạo 1 file php với nội dung:

add_action( 'my_simply_static_cron_event', 'ssp_run_static_export_cron' );
/**
 * Run Simply Static with cron every hour.
 *
 * @return void
 */
function ssp_run_static_export_cron() {
  // Full static export
  $simply_static = Simply_Static\Plugin::instance();
  $simply_static->run_static_export();
}

Chọn Run snippet everywhere như mặc định

Dùng Wp-Crontrol tạo 1 cron có Hook Name là “my_simply_static_cron_event“

Set như hình là 1 ngày chạy 1 lần, lúc 1h sáng 😀

Các bước cài đặt khá nhiều, nhưng sau khi làm các bước như trên, ta có trang static.bibica.net (GitHub Pages) và cloudflare.bibica.net (Cloudflare Pages), dù là 1 trang tĩnh, nhưng dữ liệu vẫn sẽ được tự cập nhập mới hàng ngày 😀

Trong trường hợp chỉ cần cập nhập lại nội dung 1 bài viết cụ thể, sau khi Update xong có thể ấn thẳng vào tùy chọn Generate static bên trong chỗ viết bài, nó sẽ tự cập nhập riêng bài viết và các link liên quan tới bài viết đó 😛

Cập nhập thực tế từ Simply Static Pro -> GitHub Pages, nếu chỉ cập nhập nội dung 1 bài viết cụ thể, nó rất nhanh, tầm 1-2 phút là ở GitHub đã hiện ra thay đổi 😛

Simply Static Pro -> GitHub Pages -> Cloudflare PagesL: làm đúng như trên thì thời gian cập nhập cũng khá nhanh

Tạm chưa thấy lỗi gì đáng kể ở bản v1.2.1
